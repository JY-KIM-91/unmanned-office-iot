# 서비스 카탈로그 (Service Catalog)
**무인 오피스 IoT 시스템 제공 서비스 정의**

---

## 문서 목적

이 문서는 무인 오피스 IoT 시스템이 제공하는 **4가지 핵심 서비스**를 정의한다.
각 서비스는 **기능(Function)**과 **제공 가치(Value)**를 명확히 구분하여 작성한다.

**핵심 원칙:**
- 모든 서비스는 독립형, 프리미엄, 전층형 오피스에 모두 적용 가능
- 기능은 "시스템이 무엇을 하는가"에 집중
- 가치는 "사용자/운영자에게 어떤 이익을 주는가"에 집중

---

## 서비스 구조 Overview

```
무인 오피스 IoT 시스템
│
├─ [1] 출입 인증 서비스 (Access Service)
│   └─ 누가, 언제, 어디에 접근할 수 있는가
│
├─ [2] 자원 예약 서비스 (Resource Service)
│   └─ 공간/장비를 어떻게 예약하고 이용하는가
│
├─ [3] 환경 에너지 서비스 (Environment Service)
│   └─ 조명/온도/공기질을 어떻게 제어하는가
│
└─ [4] 운영 관제 서비스 (Admin Service)
    └─ 관리자가 무엇을 보고 제어할 수 있는가
```

---

## 서비스 1: 출입 인증 서비스 (Access Service)

### 서비스 설명
**"누가, 언제, 어디에 접근할 수 있는가"**를 제어하는 서비스.
건물/지점/층/구역/실 단위의 위계적 권한 관리.

---

### 1.1. 접근 권한 위계 (Access Hierarchy)

#### 위계 구조 정의

```
Level 1: 건물 진입 (Building Entry)
  ├─ Public Zone: 메인 출입구, 로비
  │
  └─ Level 2: 층 진입 (Floor Entry)
      ├─ Shared Zone: 복도, 라운지, 팬트리
      │
      └─ Level 3: 구역 진입 (Zone Entry)
          ├─ Reservation Zone: 회의실, 포커스룸
          │
          └─ Level 4: 실 진입 (Room Entry)
              └─ Private Zone: 개별 사무실, 팀룸
```

#### 권한 상속 원칙

**상위 권한은 하위 권한을 자동 포함:**
- Level 4 권한 보유 → Level 1~3 자동 허용
- Level 2 권한만 보유 → Level 3~4 불가

**예시:**
- ABC사 직원 (Level 4: 5층 Zone-3 Private):
  - 1층 출입 ✓
  - 5층 복도 ✓
  - 5층 Zone-3 ✓
  - 5층 Zone-1 ✗ (다른 회사 공간)

- 방문객 (Level 2: 5층 복도만):
  - 1층 출입 ✓
  - 5층 복도 ✓
  - 회의실 ✗ (예약 필요)
  - Private Zone ✗

---

### 1.2. 사용자 유형별 인증 방식

#### A. 입주사 정규 직원 (Full Member)

**인증 수단:**
- 실물 카드 (RFID/NFC)
- 모바일 앱 (Bluetooth/NFC)
- 생체 인증 (선택, Phase 2)

**권한 범위:**
- Level 4 (Private Zone) 포함 전체 접근

**특징:**
- 24시간 출입 가능 (단, 회사 정책에 따라 제한 가능)
- 멤버십 등급에 따라 타 지점 로밍 가능
- 회의실 예약 가능

**시스템 동작:**
```
1. 직원이 카드 태그
2. 시스템 검증:
   - 카드 유효성 확인
   - 계약 기간 내인가? (2026-02-01 ~ 2027-01-31)
   - 현재 시각이 허용 시간대인가?
   - 해당 공간 권한이 있는가?
3. 모든 조건 충족 → 문 열림 (1초 이내)
4. 출입 기록 저장
```

---

#### B. 방문객 (Visitor)

**인증 수단:**
- QR 코드 (1회용, 시간 제한)
- 임시 PIN 번호 (선택)

**권한 범위:**
- Level 1 (Public Zone) + Level 2 (Shared Zone)
- 특정 회의실 (예약자가 지정 시)

**유형 구분:**

| 방문 유형 | 권한 | QR 발급자 | 유효 시간 |
|----------|------|----------|----------|
| 일반 방문객 | Public + Shared | CM | 당일 09:00~18:00 |
| 회의실 방문객 | Public + 특정 회의실 | 입주사 직원 | 예약 시간 ±30분 |
| 배달원 | Public (로비만) | 자동 발급 | 15분 |
| 수리 기사 | CM 지정 구역 | CM | 작업 시간 |

**시스템 동작 (일반 방문객):**
```
1. 입주사 직원이 앱에서 "손님 초대" 클릭
2. 방문객 정보 입력 (이름, 연락처, 방문 날짜)
3. 시스템이 QR 자동 생성:
   - 고유 ID 부여
   - 유효 시간: 당일 09:00~18:00
   - 권한: Public + Shared Zone
4. 방문객 휴대폰으로 QR 발송 (SMS/카카오톡)
5. 방문객이 1층에서 QR 스캔 → 문 열림
6. 출입 기록 저장 (누가 누구를 초대했는지)
```

**시스템 동작 (회의실 방문객):**
```
1. 입주사 직원이 회의실 예약 시 "외부 손님 초대" 옵션 선택
2. 방문객 정보 + 회의실 정보 입력
3. 시스템이 특별 QR 생성:
   - 유효 시간: 예약 시간 -5분 ~ +30분
   - 권한: Public + 해당 회의실만
4. 방문객이 회의실 문 앞에서 QR 스캔 → 회의실 문 열림
5. 예약 시간 외에는 출입 불가
```

---

#### C. 배달원 (Delivery)

**인증 수단:**
- 1회용 QR 코드 (자동 발급)
- 또는 무인증 (로비까지만 허용 시)

**권한 범위:**
- Level 1 (Public Zone) 로비까지만

**시스템 동작:**
```
1. 건물 외부 배달함 또는 로비 키오스크에서:
   - "배달 도착" 버튼 클릭
   - 수령인 선택 (ABC사, XYZ사 등)
   - 자동으로 15분 유효 QR 발급
2. 배달원이 QR 스캔 → 로비까지 입장
3. 배달함에 물품 보관
4. 수령인에게 앱 알림: "택배 도착"
5. 15분 경과 시 QR 자동 무효화
```

**장점:**
- CM 개입 불필요
- 보안 유지 (로비까지만)
- 수령인 실시간 알림

---

#### D. 수리/청소 업체 (Maintenance)

**인증 수단:**
- CM이 발급한 임시 카드
- 또는 시간 제한 QR

**권한 범위:**
- CM이 지정한 특정 구역만
- 예: "5층 Zone-3 회의실만 접근 가능"

**시스템 동작:**
```
1. CM이 관리자 앱에서 "임시 권한 발급"
2. 설정:
   - 대상: 수리 기사 홍길동
   - 구역: 5층 Zone-3 회의실 A
   - 시간: 2026-02-17 14:00~16:00
3. 임시 QR 발급 또는 카드 활성화
4. 수리 기사가 해당 구역만 접근 가능
5. 다른 구역 시도 시 "권한 없음"
6. 16:00 이후 자동 권한 회수
```

---

#### E. 관리자 (CM, HQ Manager)

**인증 수단:**
- 관리자 전용 마스터 카드
- 관리자 앱 (원격 제어)

**권한 범위:**
- 전체 Zone 접근 (Level 1~4 모두)
- 원격으로 모든 문 열기/잠그기 가능
- 비상 모드 활성화 권한

**시스템 동작:**
```
1. CM이 마스터 카드로 어떤 문이든 열 수 있음
2. 원격에서 "강남점 5층 Zone-3 문 열기" 클릭 시:
   - 즉시 도어락 해제
   - 해당 구역 조명 자동 점등
   - 기록 저장: "CM 김매니저가 원격 개방"
3. 비상 상황 시 "전체 문 개방" 버튼 클릭:
   - 모든 Zone의 모든 문 즉시 개방
   - 전체 회원에게 푸시: "비상 상황 발생, 대피하세요"
```

### 1.3. 멤버십 등급별 차등 권한

| 등급 | 출입 가능 지점 | 출입 시간 | 회의실 예약 | 타 지점 로밍 |
|------|--------------|----------|------------|------------|
| Premium All Access | 전국 60개 | 24시간 | 무제한 | ✓ 무제한 |
| Standard All Access | 전국 60개 | 평일 06~23시 | 월 10회 | ✓ 평일만 |
| Single Spot | 계약 지점만 | 24시간 | 월 5회 | ✗ 1회 이용권 구매 시 |
| Hot Desk | 계약 지점만 | 예약 시간만 | 불가 | ✗ |

**시스템 동작 (타 지점 로밍 시도):**
```
Single Spot 회원이 홍대점 출입 시도 (계약은 강남점):
1. 카드 태그
2. 시스템 검증:
   - 멤버십 등급: Single Spot
   - 계약 지점: 강남점
   - 현재 위치: 홍대점 → 불일치
3. 단말기 화면:
   - "계약 지점이 아닙니다"
   - "1회 이용권 구매 시 이용 가능 (2만원)"
   - "지금 구매하기" 버튼
4. 회원이 앱에서 결제 완료 시:
   - 당일 홍대점 출입 권한 즉시 부여
   - 자정에 자동 권한 회수
```

---

### 1.4. 시간대별 차등 권한

#### 일반 시간대 (평일 06:00~22:00)
- 모든 회원 출입 가능
- 방문객 QR 자동 발급 가능

#### 야간 시간대 (평일 22:00~06:00)
- Manager 등급 이상만 출입 가능
- 일반 회원 출입 시도 시:
  - "야간 출입 제한 시간입니다"
  - "긴급 출입이 필요하면 CM에게 연락하세요"
- CM에게 야간 출입 시도 알림 발송
- CM이 원격 승인 시 임시 권한 부여

#### 주말/공휴일
- 회사별 정책에 따라 다름
- 기본: 출입 가능하나 환경 제어는 절전 모드
- 설정: 주말 출입 완전 금지 가능

---

### 1.5. 제공 가치 (Access Service Value)

#### 사용자 측 가치

**입주사 직원:**
- ✅ **편의성**: 카드 하나로 모든 공간 접근 (다중 키 불필요)
- ✅ **유연성**: 24시간 출입 가능 (야근, 주말 근무 자유)
- ✅ **안심감**: 미등록자 자동 차단으로 보안 강화

**방문객:**
- ✅ **간편함**: 휴대폰 QR만으로 건물 진입 (별도 등록 불필요)
- ✅ **명확함**: 출입 가능 시간/구역이 QR에 명시됨
- ✅ **즉시성**: 초대장 받은 즉시 사용 가능

**관리자 (CM):**
- ✅ **자동화**: 방문객 등록 수동 처리 불필요 (월 100건 → 0건)
- ✅ **제어력**: 원격에서 언제든 권한 부여/회수 가능
- ✅ **투명성**: 누가 언제 출입했는지 100% 기록

#### 운영사 측 가치

**보안 강화:**
- 미등록자 출입 차단율 100%
- 출입 기록 100% 저장 (감사 추적 가능)
- 이상 징후 자동 탐지 (3회 실패, 카드 복제 의심 등)

**비용 절감:**
- 실물 키 관리 불필요 (분실/복제 리스크 제거)
- CM 업무 시간 월 25시간 절약 (방문객 등록)
- 보안 사고 감소 → 보험료 절감

**확장성:**
- 신규 회원 등록 10분 이내 완료
- 퇴사자 권한 즉시 회수 (실물 키 회수 불필요)
- 지점 추가 시 권한 정책 자동 적용

## 서비스 2: 자원 예약 서비스 (Resource Service)

### 서비스 설명
**"공간/장비를 어떻게 예약하고 이용하는가"**를 관리하는 서비스.
회의실, 좌석, 주차장 등 제한된 자원의 효율적 배분.

---

### 2.1. 예약 가능 자원 유형

#### A. 공용 회의실 (Shared Meeting Room)

**특징:**
- 전체 입주사가 공유하는 자원
- **예약 필수** (예약 없이 사용 불가)
- **과금 대상** (시간당 요금 부과)
- 초과 이용 시 추가 과금

**시스템 동작:**
```
1. 회원이 앱에서 "회의실 예약" 메뉴
2. 날짜/시간 선택 (2026-02-17 14:00~16:00)
3. 회의실 선택 (강남점 회의실 A)
4. 예약 정보 입력:
   - 회의 제목 (선택)
   - 참석 인원
   - 외부 손님 초대 여부
5. 요금 확인 및 결제:
   - 2시간 x 2만원 = 4만원
   - 멤버십 무료 쿼터 사용 여부 확인
6. 예약 확정 시:
   - 예약 QR 자동 발급
   - 예약 시작 5분 전 푸시 알림
   - 회의실 도어락에 권한 자동 부여
7. 예약 시간 도래:
   - 13:55부터 예약자 카드로 문 열림
   - 회의실 내부 화면에 예약 정보 표시
```

**과금 방식:**
- 기본료: 시간당 2만원 (회의실에 따라 차등)
- 초과 이용료: 10분당 3,500원
- 멤버십 무료 쿼터:
  - Premium: 무제한
  - Standard: 월 10회
  - Single Spot: 월 5회

---

#### B. 내부 미팅룸 (Private Meeting Room)

**특징:**
- 특정 회사 전용 공간
- **예약 선택** (예약 없이도 사용 가능)
- **과금 없음** (회사 월 요금에 포함)
- 단, 예약 시스템으로 사용 현황 관리

**시스템 동작:**
```
1. ABC사 직원이 내부 미팅룸 사용 희망
2. 앱에서 "내부 미팅룸 현황" 확인:
   - 미팅룸 1: 사용 중 (09:00~11:00 김대리 예약)
   - 미팅룸 2: 사용 가능 ✓
3. 두 가지 옵션:
   A. 예약하기 (미팅룸 2, 14:00~15:00)
      → 다른 직원이 예약 현황 확인 가능
   B. 바로 사용 (예약 없이 입장)
      → 문 열리고, 자동으로 "즉시 사용 중" 상태 표시
4. 다른 직원이 미팅룸 2 문 앞 도착:
   - 단말기 화면: "현재 사용 중 (이과장)"
   - 예상 종료 시간: 15:00
5. 사용 종료 시:
   - 퇴실 감지 → 자동으로 "사용 가능" 상태 전환
```

**공용 회의실과의 차이:**

| 구분 | 공용 회의실 | 내부 미팅룸 |
|------|-----------|-----------|
| 예약 | 필수 | 선택 |
| 과금 | 시간당 요금 | 무료 (월 요금 포함) |
| 무단 사용 | 차단 (문 안 열림) | 허용 (문 열림) |
| 초과 이용 | 추가 과금 | 과금 없음 |
| 노쇼 처리 | 자동 취소 + 페널티 | 페널티 없음 |
| 외부 손님 | 초대 가능 | 회사 직원만 |

---

### 2.2. 예약 프로세스

#### 예약 생성 (Booking Creation)

**정보 입력:**
- 자원 유형 (회의실/좌석/주차장)
- 날짜 및 시간
- 인원 (선택)
- 외부 손님 초대 (선택)

**예약 가능 여부 검증:**
```
1. 선택한 시간대에 기존 예약 없는가?
2. 회원의 예약 쿼터가 남아 있는가?
3. 연속 예약 제한 (4시간 이상 불가) 위반 안 했는가?
4. 과거 노쇼 이력으로 인한 제재 중이 아닌가?
```

**예약 확정:**
- 예약 ID 발급
- QR 코드 생성
- 캘린더에 자동 등록
- 이메일/앱 알림 발송

---

#### 예약 변경 (Booking Modification)

**변경 가능 항목:**
- 시간 연장/단축
- 인원 변경
- 외부 손님 추가/삭제

**변경 제약:**
- 예약 시작 1시간 전까지만 변경 가능
- 다음 예약이 있으면 연장 불가
- 시간 단축 시 환불 정책 적용

**시스템 동작:**
```
1. 회원이 예약 상세 화면에서 "변경" 버튼
2. 변경 내용 입력 (14:00~16:00 → 14:00~17:00)
3. 시스템 검증:
   - 16:00~17:00 시간대 예약 없음 확인
   - 추가 요금 계산 (1시간 = 2만원)
4. 결제 완료 시 예약 업데이트
5. 회의실 도어락 권한 자동 연장 (16:00 → 17:00)
```

---

#### 예약 취소 (Booking Cancellation)

**환불 정책:**
- 예약 시작 3시간 전 취소: 100% 환불
- 예약 시작 1시간 전 취소: 50% 환불
- 예약 시작 1시간 이내 취소: 환불 불가

**시스템 동작:**
```
1. 회원이 예약 취소 버튼
2. 환불 금액 안내
3. 확인 시:
   - 예약 상태 "취소됨"으로 변경
   - 회의실 도어락 권한 자동 회수
   - 환불 처리 (영업일 기준 3일)
   - 예약 슬롯 "예약 가능" 상태로 전환
```

### 2.3. 노쇼(No-Show) 자동 대응 로직

#### 노쇼 정의
예약 시작 후 15분 이내 입실하지 않은 경우.

#### 감지 방법
```
재실 감지 센서 (회의실 내부)
  +
도어락 개폐 이력
  =
실제 입실 여부 판단
```

#### 자동 대응 프로세스

**Step 1: 예약 시작 (14:00)**
```
- 예약자에게 푸시: "회의실 A 예약 시작 5분 전입니다"
- 회의실 도어락 권한 활성화 (13:55부터)
```

**Step 2: 5분 경과 (14:05)**
```
- 재실 감지 센서 확인
- 입실 없음 → 대기
```

**Step 3: 10분 경과 (14:10)**
```
- 여전히 입실 없음
- 예약자에게 푸시: "회의실 A 예약 확인 부탁드립니다. 5분 내 입실하지 않으면 자동 취소됩니다"
```

**Step 4: 15분 경과 (14:15)**
```
- 여전히 입실 없음 → 노쇼 확정
- 시스템 자동 처리:
  1. 예약 상태 "노쇼"로 변경
  2. 도어락 권한 회수
  3. 예약 슬롯 "예약 가능"으로 전환
  4. 예약자에게 알림:
     - "회의실 A 노쇼로 예약 취소되었습니다"
     - "예약료는 환불되지 않습니다"
     - "노쇼 3회 누적 시 1주일 예약 제한"
  5. 노쇼 이력 기록 (회원 프로필에 저장)
  6. CM에게 알림: "회의실 A 노쇼 발생, 현재 예약 가능 상태"
```

**Step 5: 다른 회원에게 즉시 예약 가능 알림 (선택)**
```
- 같은 지점 회원들에게 푸시:
  - "회의실 A 14:15~16:00 예약 가능합니다"
  - "지금 예약하기" 버튼
```

---

#### 노쇼 페널티 정책

| 노쇼 횟수 | 페널티 | 복구 조건 |
|----------|--------|----------|
| 1회 | 경고 메시지 | - |
| 2회 | 다음 예약 시 경고 팝업 | - |
| 3회 | 1주일 예약 제한 | 7일 경과 |
| 5회 | 1개월 예약 제한 | 30일 경과 |
| 10회 | 예약 권한 영구 정지 | CM 승인 필요 |

**노쇼 이력 리셋:**
- 연속 10회 정상 이용 시 노쇼 카운트 1회 차감
- 매년 1월 1일 전체 리셋

---

#### 노쇼 방지 기능

**사전 알림:**
- 예약 1일 전: "내일 회의실 A 예약이 있습니다"
- 예약 1시간 전: "1시간 후 회의실 A 예약입니다"
- 예약 5분 전: "5분 후 예약 시작입니다"

**원클릭 취소:**
- 알림에 "취소하기" 버튼 포함
- 클릭 시 즉시 취소 + 환불 정책 적용

**자동 체크인:**
- 회의실 문 앞 단말기에 "체크인" 버튼
- 예약 시작 전에 미리 체크인 가능
- 체크인 시 "입실 확인됨" 상태로 변경 (노쇼 방지)

### 2.4. 초과 이용 자동 감지 및 과금

#### 감지 방법
```
예약 종료 시각 (16:00)
  ↓
재실 감지 센서 확인
  ↓
사람 감지 O → 초과 이용 시작
  ↓
10분 단위로 카운트
```

#### 자동 대응 프로세스

**예약 종료 10분 전 (15:50)**
```
- 회의실 내부 스피커: "예약 종료 10분 전입니다"
- 회의실 화면: "10분 후 종료 / 연장하기" 버튼 표시
```

**예약 종료 시각 (16:00)**
```
- 회의실 화면:
  - "예약 시간이 종료되었습니다"
  - "퇴실 버튼을 눌러주세요"
  - "연장 사용 시 10분당 3,500원"
- 앱 푸시: "회의실 A 예약 종료. 퇴실 또는 연장하세요"
```

**초과 10분 (16:10)**
```
- 재실 감지 계속됨
- 시스템 자동 처리:
  1. 초과 이용 시작 기록
  2. 초과 시간: 10분
  3. 추가 요금: 3,500원
  4. 앱 푸시: "10분 초과 이용 중 (3,500원 청구 예정)"
  5. 회의실 화면: "초과 10분 / 3,500원"
```

**초과 20분 (16:20)**
```
- 앱 푸시: "20분 초과 이용 중 (7,000원 청구 예정)"
- 회의실 화면: "초과 20분 / 7,000원"
```

**퇴실 감지 (16:25)**
```
- 재실 감지 센서: 사람 없음
- 도어락 개폐 이력: 16:24 퇴실
- 시스템 자동 처리:
  1. 최종 초과 시간: 25분 → 30분 단위 반올림
  2. 최종 추가 요금: 3,500원 x 3회 = 10,500원
  3. 앱 푸시: "회의실 A 25분 초과 이용. 추가 요금 10,500원이 다음 청구서에 합산됩니다"
  4. 빌링 시스템에 자동 기록
  5. 회의실 상태 "예약 가능"으로 전환
  6. 조명 자동 소등 (1분 후)
```

---

#### 연장 기능

**앱에서 연장:**
```
1. 예약 종료 10분 전부터 "연장하기" 버튼 활성화
2. 연장 가능 여부 자동 확인:
   - 다음 예약 없음 → 30분/1시간 연장 가능
   - 다음 예약 있음 (16:30 시작) → "연장 불가" 안내
3. 연장 시간 선택 및 결제
4. 결제 완료 시:
   - 예약 시간 자동 연장
   - 도어락 권한 연장
   - 회의실 화면 업데이트 (종료 시각 16:30으로 변경)
```

### 2.5. 제공 가치 (Resource Service Value)

#### 사용자 측 가치

**입주사:**
- ✅ **가시성**: 실시간 회의실 사용 현황 확인 (앱에서 한눈에)
- ✅ **공정성**: 선착순 예약으로 공평한 자원 배분
- ✅ **투명성**: 초과 이용 요금 실시간 확인 (깜빡 초과 방지)

**관리자 (CM):**
- ✅ **자동화**: 예약 관리 수동 개입 불필요
- ✅ **효율**: 노쇼 자동 처리로 공간 낭비 최소화
- ✅ **수익**: 초과 이용 자동 과금으로 수익 누락 제로

#### 운영사 측 가치

**공간 활용 극대화:**
- 노쇼 자동 취소로 예약률 90% → 실제 이용률 85%
- 빈 공간 즉시 다른 회원에게 제공
- 회의실당 일평균 이용 시간 20% 증가

**수익 증대:**
- 초과 이용 수익 누락 0% (기존 30% 누락)
- 연간 추가 매출 약 3,600만원 (60개 지점)

**데이터 기반 의사결정:**
- 회의실별 예약률 분석 → 인기 시간대 파악
- 노쇼율 높은 회원 패턴 분석
- 공간 추가/축소 의사결정 근거

### 3.2. Smart Lighting (조명 자동 제어)

#### 구역별 독립 제어

**Private Zone (개별 사무실):**
```
사무실을 3~5개 구역으로 분할:
- Zone-3A: 입구 구역
- Zone-3B: 업무 구역
- Zone-3C: 휴게 구역

각 구역마다 독립된 재실 감지 센서:
- 3B에 사람 있음 → 3B만 조명 켜짐
- 3A, 3C 비어 있음 → 소등
```

**Shared Zone (라운지):**
```
- 재실 감지 시 조명 즉시 점등
- 30분간 재실 없음 → 자동 소등
- 낮 시간대 (12:00~14:00) → 자연광 충분 시 조명 50% 밝기
```

**Reservation Zone (회의실):**
```
- 예약 시작 5분 전 → 조명 자동 점등
- 예약 종료 + 퇴실 감지 → 1분 후 소등
- 예약 없을 시 조명 소등 유지
```

---

#### 시나리오: 첫 출근 시 순차 점등

**상황:** 오전 8시, ABC사 첫 출근자 김대리

**시스템 동작:**

#### 자동 환기 제어

**재실 인원 기반:**
```

재실 5~9명 → 환기 70% 가동
재실 1~4명 → 환기 50% 가동
재실 0명 → 환기 10% 가동 (최소 순환)
```

**공기질 기반:**
```
CO2 1000ppm 이상 → 환기 강제 100%
미세먼지 나쁨 (외부) → 환기 중단, 공기청정기 가동
미세먼지 좋음 (외부) → 창문 자동 개방 (선택, Phase 2)
```

### 4.3. 비상 상황 Fail-Safe 로직

#### 화재 경보 연동

**자동 트리거:**
```
화재 경보기 신호 감지
  ↓
시스템이 자동으로 비상 모드 활성화
  ↓
모든 도어락 즉시 해제
  ↓
대피 안내 메시지 표시
```

**상세 동작:**
```
1. 화재 경보기 신호 수신 (건물 방재 시스템 연동)
2. 0.5초 이내 시스템 자동 판단:
   - 진짜 화재인가?
   - 오작동인가?
   - 판단: 일단 비상 모드 활성화 (안전 우선)
3. 즉시 처리:
   - 모든 Zone의 모든 도어락 해제
   - 승강기 제어 시스템에 신호 (1층 자동 하강)
   - 비상등 자동 점등
   - 비상 방송 자동 재생: "화재 경보 발생. 침착하게 대피하세요"
4. 알림 발송:
   - 현재 재실자 전원에게 앱 푸시
   - CM에게 긴급 전화 자동 발신
   - 본사 운영팀에게 긴급 알림
   - 119 자동 신고 (Phase 2)
5. 대피 완료 확인:
   - 재실 센서로 인원 감소 모니터링
   - CM이 "대피 완료" 확인 후 비상 모드 해제
```

---

#### 정전 시 배터리 백업

**자동 동작:**
```
정전 감지
  ↓
자동으로 배터리 전원 전환 (0.1초)
  ↓
4시간 동안 정상 작동
  ↓
배터리 20% 이하 시 CM 알림
  ↓
배터리 5% 이하 시 "안전 모드" (문 자동 개방)
```

**안전 모드:**
```
배터리 5% 이하 도달 시:
1. 모든 도어락 자동 해제 (사람 가두지 않기 위함)
2. 단말기 화면: "배터리 부족, 문이 개방됩니다"
3. CM에게 긴급 알림: "배터리 고갈 임박"
4. 재실자에게 앱 푸시: "정전으로 인한 배터리 모드. 안전하게 퇴실하세요"
```

- 원격 제어 불가 (로컬 단말기만)

### 4.4. 제공 가치 (Admin Service Value)

#### 관리자 (CM) 측 가치

**가시성:**
- ✅ 실시간으로 지점 전체 상황 파악 (사무실 안 가도 됨)
- ✅ 이상 징후 즉시 알림 (수동 점검 불필요)
- ✅ 에너지 사용량 실시간 확인 (절감 효과 가시화)

**제어력:**
- ✅ 원격에서 모든 문 제어 (현장 출동 불필요)
- ✅ 권한 즉시 부여/회수 (카드 분실 시 빠른 대응)
- ✅ 비상 상황 즉시 대응 (버튼 한 번으로 전체 제어)

**효율성:**
- ✅ 수동 점검 시간 제로 (자동 모니터링)
- ✅ 장치 고장 사전 감지 (배터리 부족 알림)
- ✅ 월말 리포트 자동 생성 (수동 작성 불필요)

#### 운영사 (HQ) 측 가치

**통합 관제:**
- ✅ 60개 지점 한 화면에서 관리
- ✅ 지점별 비교 분석 (점유율, 에너지 효율)
- ✅ 전사 정책 15분 만에 배포 (기존 25시간)

**의사결정 지원:**
- ✅ 데이터 기반 공간 최적화 (인기 구역 파악)
- ✅ 에너지 절감 효과 정량 측정
- ✅ 보안 사고 추이 분석

**리스크 관리:**
- ✅ 비상 상황 자동 대응 (화재 시 자동 개방)
- ✅ 보안 사고 사전 차단 (이상 징후 탐지)
- ✅ 감사 추적 100% (모든 기록 저장)

## 서비스 간 연동 시나리오

### 시나리오 1: 회의실 예약 → 출입 + 환경 자동 제어

```
1. [Resource] 회원이 회의실 A 예약 (14:00~16:00)
   ↓
2. [Access] 예약자 카드에 자동 권한 부여 (13:55~16:00)
   ↓
3. [Environment] 13:40부터 사전 냉난방 가동
   ↓
4. [Environment] 13:55에 조명 자동 점등
   ↓
5. [Access] 14:00에 예약자 카드 태그 → 문 열림
   ↓
6. [Resource] 16:00 예약 종료 → [Access] 권한 회수
   ↓
7. [Environment] 퇴실 감지 → 1분 후 조명 소등, 냉난방 중단
   ↓
8. [Admin] CM에게 "회의실 A 청소 필요" 알림
```

---

### 시나리오 2: 야간 출입 시도 → 보안 알림

```
1. [Access] 23:30 김직원 카드 태그 (야간 시간대)
   ↓
2. [Access] 권한 검증: 일반 회원 → 야간 출입 제한
   ↓
3. [Admin] CM에게 즉시 알림: "야간 출입 시도 (김직원)"
   ↓
4. CM이 상황 확인 후 원격 승인
   ↓
5. [Access] 임시 권한 부여 → 문 열림
   ↓
6. [Environment] 김직원 구역만 조명 점등
   ↓
7. [Admin] 출입 기록 저장 (야간 출입 특별 표시)
```

---

### 시나리오 3: 마지막 퇴근 → 전체 자동 절전

```

   ↓
2. [Environment] 재실 센서: 0명 감지
   ↓
3. [Environment] 10분 후 (23:00):
   - 전체 조명 자동 소등
   - 냉난방 자동 중단
   - 환기 최소 모드
   ↓
4. [Admin] 에너지 사용량 급감 기록
   ↓
5. 익일 08:30:
   - [Environment] 출근 패턴 학습 기반 자동 냉난방 가동
   - [Admin] 에너지 절감 효과 리포트 업데이트
```

## 서비스 확장성 (Scalability)

### 독립형 → 프리미엄 → 전층형 모두 지원

| 서비스 | 독립형 (소규모) | 프리미엄 (중규모) | 전층형 (대규모) |
|--------|----------------|------------------|----------------|
| **Access** | 5개 Zone | 15개 Zone | 전체 층 단일 Zone |
| **Resource** | 회의실 2개 | 회의실 5개 | 회의실 10개 + 내부 미팅룸 5개 |
| **Environment** | 층별 제어 | 구역별 독립 제어 | 회사별 맞춤 제어 |
| **Admin** | 지점 CM 관리 | 지점 CM + 본사 모니터링 | 본사 통합 관제 |

**핵심:** 시스템 구조는 동일, 단지 Zone 수와 장치 수만 증가.
