## 1. 6개 축 비교 분석

### 1.1 출입 시스템 방식

| 기업 | 출입 방식 | 기술 스택 | 정보 출처 |
|------|----------|----------|----------|
| **WeWork** | 카드 리더 + 모바일 앱 (추정) | CCTV 통합 출입통제 시스템, 글로벌 표준 벤더 사용 추정 | ✓ 확인됨 |
| **스파크플러스** | 모바일 QR 코드 중심 | QR 기기 태그 방식, 외부 방문자 QR 전송 초대 | ✓ 확인됨 |
| **패스트파이브** | 모바일 앱 (블루투스 + 위치 기반) | AirFob Space 도입, RFID → 모바일 전환 완료 | ✓ 확인됨 |

**주요 발견**:
- 패스트파이브와 스파크플러스는 모바일 중심 출입 시스템 구축 완료
- WeWork는 전통적 카드 리더 + 모바일 혼합 방식으로 추정
- 패스트파이브의 블루투스 기반 방식이 가장 사용자 편의성 높음

**가설**:
- WeWork는 글로벌 표준화를 위해 검증된 벤더(Kisi, Brivo 등) 사용 가능성 높음
- 국내 업체들은 자체 개발 또는 국내 벤더(AirFob 등) 활용

### 1.2 운영 자동화 수준

| 기업 | 자동화 수준 | 핵심 기능 | 자체 개발 여부 |
|------|------------|----------|--------------|
| **WeWork** | 부분 무인 | 예약, 방문자 관리, 캘린더 통합, 실시간 공간 분석 | △ Yardi 파트너십 (공동 개발) |
| **스파크플러스** | 부분 무인 | 출입, 회의실 예약, 계약/정산 자동화, 시간대별 자동화 설정 | ✓ 자체 개발 OS 시스템 (2022) |
| **패스트파이브** | 부분 무인 | 모바일 출입, 회의실/스튜디오 예약, 자동 인증서 만료 | ✓ 자체 개발 + 세일즈포스 통합 |

**주요 발견**:
- 모든 업체가 부분 무인 운영 (완전 무인은 미구현)
- 스파크플러스와 패스트파이브는 자체 OS 보유
- WeWork는 SaaS 플랫폼 파트너십 전략

**가설**:
- 완전 무인 운영은 보안/안전 이슈로 업계 표준이 아님
- 국내 업체들이 자체 개발 역량에서 WeWork 대비 우위 가능성

**전략적 시사점**:

- WeWork 플랫폼은 타 빌딩 판매까지 고려한 확장성 설계

**가설**:
- 국내 업체들은 아웃소싱 보안 업체 활용 가능성
- 패스트파이브의 중앙화 시스템은 통합 관제 기반 마련된 것으로 추정

## 2. 전략적 시사점

### 2.1 패스트파이브의 강점

1. **운영 모델 다변화 선도**
   - 위탁/RS/사옥 솔루션 등 다양한 계약 구조 지원
   - 초기 투자 부담 완화하면서 시장 확장 중

2. **하이브리드 기술 전략**
   - 출입: 검증된 외부 벤더 (AirFob) 활용
   - 운영: 자체 개발 + Salesforce CRM 통합
   - 리스크 분산과 비용 최적화 균형

3. **중앙화 시스템 구축 완료**
   - Salesforce 기반 통합 CS 체계
   - 전 지점 표준화된 고객 경험 제공 가능

### 2.2 출입 시스템 내재화 시 고려사항

#### 장점 (Pros)
- **벤더 독립성 확보**: 장기 구독 비용 절감 (도어당 월 $50-150 × 지점 수)
- **커스터마이징 자유도**: 다변화된 운영 모델에 맞춤 기능 개발 가능
- **기술 자산화**: 자체 플랫폼으로 경쟁 우위 강화
- **확장성**: 파워드바이 모델 등 B2B 솔루션 판매 가능

#### 단점 (Cons)
- **초기 개발 비용**: R&D 인력, 하드웨어 개발, 테스트 비용
- **유지보수 부담**: 지속적인 기술 지원팀 필요
- **검증 기간**: 안정화까지 6-12개월 소요 예상
- **하드웨어 의존**: 컨트롤러, 리더 등 물리 장비는 외주 불가피

#### 리스크
- 보안 사고 발생 시 책임 소재 명확 (자체 개발 시 모든 책임)
- 기술 트렌드 변화 대응 (생체인식, AI 기반 보안 등)
- 다수 지점 동시 전환 시 운영 리스크

### 2.3 권장 접근 방식

**Phase 1: 하이브리드 모델 유지 (현재)**
- AirFob 등 검증된 벤더 활용 지속
- 자체 운영 플랫폼 고도화에 집중
- 출입 API 통합 수준 강화

**Phase 2: 단계적 내재화 검토**
- 파일럿 지점 (1-2개) 자체 개발 시스템 테스트
- 벤더 비용 vs 자체 개발 비용 실증 데이터 확보
- 보안 인증 (ISO 27001 등) 사전 준비

**Phase 3: 전략적 의사결정**
- ROI 분석 기반 전면 내재화 vs 하이브리드 유지 결정
- B2B 솔루션 판매 가능성 검토 (파워드바이 모델 확장)
