## 1. 제안 개요 (Executive Summary)

### 1.1 프로젝트명

**패스트파이브 통합 공간 관제 솔루션 (Space OS)**

> **한 줄 요약**: 파편화된 외부 벤더 시스템을 하나의 통합 관제 화면으로 묶고, 산업용 스펙의 자체 하드웨어로 무중단 안정성을 확보하며, 자동화된 룰 엔진으로 현장 매니저의 수동 업무와 과금 누수를 동시에 제거하는 공간 관제 플랫폼입니다.

---

### 1.2 왜 지금 해야 하는가 (Why Now)

패스트파이브는 서울/수도권 약 60개 지점, 2,600개 입주 업체, 약 3만 명의 이용자를 보유한 국내 대표 공유오피스입니다. 그러나 현재의 운영 인프라는 사업 규모의 성장 속도를 따라가지 못하고 있으며, 다음 네 가지 구조적 한계가 더 이상 방치할 수 없는 수준에 이르렀습니다.

---

#### 문제 1. 이기종 벤더 종속에 의한 비용 누적과 데이터 파편화

출입 통제(슈프리마/에어팝), 영상 감시(캡스), 공조/조명(Tuya 등)이 각기 다른 외부 벤더에 종속되어 운영되고 있습니다.

| 현상 | 비즈니스 영향 |
|------|-------------|
| 벤더별 개별 관리 콘솔 4~5개 병행 사용 | 단일 관리자 뷰(Single Pane of Glass) 부재 → 관제 사각지대 발생 |
| 시스템 간 데이터 교차 분석 불가 | 출입 로그 × 예약 데이터 × 에너지 데이터 연계 불가 → 운영 의사결정 근거 부족 |
| 지점 수 증가 시 라이선스 비용 선형 증가 | **3년 TCO(총 소유비용) 약 18.5억원** (60개 지점/500개 도어 기준) |
| 기능 커스터마이징 제한 | 벤더 로드맵에 종속 → 패스트파이브 고유의 운영 니즈 반영 불가 |

> **핵심 페인포인트**: 지점이 늘어날수록 벤더 비용은 선형으로 증가하지만, 확보할 수 있는 데이터와 제어력은 정체되어 있습니다.

---

#### 문제 2. 현장 매니저의 수동 관리 과부하

지점 매니저(CM)가 출입 권한 관리, 회의실 분쟁 중재, 설비 점검 등을 수동으로 처리하고 있습니다.

| 수동 업무 항목 | 지점당 월간 소요 시간 | 60개 지점 환산 |
|--------------|-------------------|--------------|
| 회의실 어뷰징 순찰 및 분쟁 중재 | 약 20시간 | **1,200시간/월** |
| 설비 상태 수동 점검 | 약 8시간 | 480시간/월 |
| 방문객 출입 등록 수동 처리 | 약 10시간 | 600시간/월 |
| **합계** | **약 38시간/지점** | **약 2,280시간/월** |

> **핵심 페인포인트**: CM이 단순 반복 업무에 하루 1~2시간을 소모하고 있어, 입주사 케어와 서비스 품질 향상이라는 본연의 업무에 집중할 수 없습니다.

---

#### 문제 3. 회의실 어뷰징으로 인한 매출 누수

회의실의 미예약 사용, 초과 사용 문제가 심각하지만, 현재 이를 체계적으로 탐지하고 과금할 수 있는 수단이 없습니다.

| 어뷰징 유형 | 현재 대응 방식 | 연간 추정 손실 |
|-----------|-------------|-------------|
| 예약 없이 회의실 점거 | CM이 직접 순찰하여 확인 | 측정 불가 (비가시 영역) |
| 예약 시간 초과 사용 | CM이 확인 후 구두 경고 | **약 3,600만원** (과금 누락률 30%) |
| 노쇼(No-Show) | 15분 경과 후 CM이 수동 취소 | **약 8,000만원** (공실 기회비용) |
| **합계** | | **연간 약 1.16억원 이상** |

> **핵심 페인포인트**: 과금 누수와 공실 손실이 연간 1억원 이상 발생하고 있으나, 현재 시스템으로는 정확한 측정조차 불가능합니다.

---

#### 문제 4. 내부 담당자 1인 과부하

고객사 내부에서 하드웨어 R&D, 비전 AI 테스트, 벤더 관리 등을 **담당자 1인이 모두 전담**하고 있어 극심한 병목이 발생하고 있습니다.

| 담당 영역 | 현재 상태 |
|---------|---------|
| 하드웨어 R&D (리더기, 센서, 릴레이) | 1인이 직접 설계/테스트 |
| AI 비전 PoC | 1인이 직접 모델 탐색/테스트 |
| 벤더 관리 (슈프리마, 에어팝, 캡스 등) | 1인이 개별 벤더 대응 |
| 신규 기술 검토 | 전략적 검토 시간 확보 불가 |

> **핵심 페인포인트**: "전문가 한 명"이 모든 것을 짊어진 구조에서는, 그 한 명이 이탈하면 전체 기술 기반이 무너집니다. 이것은 사업 연속성(BCP) 차원의 리스크입니다.

### 1.3 제안 접근 전략: 린(Lean) 원칙 기반 단계적 구축

> **원칙: "모든 것을 밑바닥부터 만들지 않는다."**

전체 IoT 플랫폼을 자체 개발하는 것은 막대한 초기 투자와 긴 개발 기간을 요구합니다. 본 제안은 린(Lean) 소프트웨어 개발의 핵심 원칙인 **"낭비를 제거하라"**와 **"빨리 인도하라"**에 따라, 검증된 글로벌 IoT 플랫폼과 산업 표준 프로토콜을 적극 활용하여 최소한의 투자로 최대 가치를 빠르게 전달합니다.

| 영역 | 자체 구축 시 (비효율) | Space OS 전략 (린 접근) |
|------|-------------------|----------------------|
| IoT 디바이스 통신 | 자체 프로토콜 설계 (6개월+) | **Tuya Cloud-to-Cloud API** 연동으로 1,100+ 제품군 즉시 통합 |
| CCTV 연동 | 자체 VMS 개발 (4개월+) | **ONVIF 표준** 프로토콜로 벤더 무관하게 기존 카메라 즉시 연동 |
| 엣지 컴퓨팅 | 자체 엣지 OS 개발 (8개월+) | **AWS IoT Greengrass** 기반 산업용 Edge Gateway 채택 |
| 비전 AI | 자체 데이터 수집 + 모델 훈련 | 고도화 단계에서 사전 학습 모델 활용 (Phase 2 이후 확장) |

이 전략의 핵심은 **"만들 수 있는 것"이 아니라 "검증해야 하는 것"에 집중하는 것**입니다. Phase 1에서 파일럿 지점을 통해 다음 핵심 가설을 빠르게 검증합니다.

- [ ] **가설 1**: 이기종 기기를 단일 대시보드에서 통합 제어할 수 있는가?
- [ ] **가설 2**: 센서 + 예약 데이터 교차 검증으로 회의실 어뷰징을 자동 탐지할 수 있는가?
- [ ] **가설 3**: 타워램프 등 물리적 피드백으로 어뷰징 행위를 실질적으로 억제할 수 있는가?

### 3.2 핵심 기술 전략 ①: 산업용 Edge Gateway로 무중단 운영 확보

> **"클라우드가 끊겨도 지점은 멈추지 않는다."**

공유오피스의 출입 통제와 설비 제어는 **24시간 365일 무중단**이 보장되어야 합니다. 현재 벤더 클라우드 종속 구조에서는 외부 서비스 장애 시 패스트파이브가 직접 대응할 수 있는 방법이 없습니다. Space OS는 각 지점에 산업용 스펙의 Edge Gateway를 배치하여 이 문제를 근본적으로 해결합니다.

| 설계 원칙 | 구현 방식 | 비즈니스 효과 |
|----------|---------|-------------|
| **오프라인 독립 운영** | Edge에 출입 권한 캐시(4시간 TTL) + 이벤트 버퍼(48시간) 보유 | 인터넷 단절 시에도 출입/제어 정상 동작 |
| **이중화 네트워크** | 유선 Ethernet(Primary) + 4G/LTE(Failover) | 네트워크 단일 장애점(SPOF) 제거 |
| **배터리 백업** | UPS 탑재 (4시간 독립 전원) | 정전 시에도 출입 제어 유지 |
| **원격 관리** | AWS IoT Greengrass 기반 OTA 펌웨어 업데이트 | 현장 출동 없이 소프트웨어 업데이트 |
| **자체 릴레이 보드** | ESP32-S3 기반 4채널 릴레이 (BOM 5만원/대) | 벤더 종속 없이 도어락/조명/타워램프 직접 제어 |

**Fail-safe 시나리오 보장:**

| 장애 상황 | 시스템 자동 대응 |
|---------|--------------|
| 지점 인터넷 단절 (2시간) | Edge 캐시로 출입 지속 허용, 이벤트는 로컬 큐에 저장 → 복구 시 일괄 동기화 |
| AWS 클라우드 장애 (30분) | 모든 Edge 영향 없이 독립 운영, 관제 콘솔만 일시 중단 |
| 정전 (4시간 이내) | UPS 배터리 전환 (0.1초), 배터리 5% 이하 시 도어 자동 개방(인명 안전 우선) |
| Edge Gateway 하드웨어 고장 | 본사 즉시 알림 → 예비 장비 교체 (4시간 SLA 목표) |

### 3.3 핵심 기술 전략 ②: Tuya C2C API로 이기종 기기 즉시 통합

> **"프로토콜을 새로 만들지 않는다. 이미 있는 것을 연결한다."**

Tuya IoT 플랫폼은 전 세계 200+ 카테고리, 1,100+ 제품군이 등록된 글로벌 IoT 생태계입니다. Space OS는 Tuya의 Cloud-to-Cloud API를 활용하여, 별도의 디바이스 프로토콜 개발 없이 다양한 IoT 기기를 즉시 통합합니다.

| 항목 | 내용 |
|------|------|
| **연동 방식** | Tuya Open API (REST + WebSocket 실시간 이벤트) |
| **통합 가능 기기** | 조명 스위치, 스마트 플러그, 환경 센서(온습도/CO2), 커튼 모터, 전력 모니터 등 |
| **핵심 이점** | 기기를 바꿔도 플랫폼은 그대로 → 하드웨어 선택의 자유도 확보 |
| **장기 전략** | Phase 2에서 Tuya 의존도를 낮추고 직접 MQTT 연동으로 전환 가능 (벤더 중립성 확보) |

---

### 3.4 핵심 기술 전략 ③: ONVIF 표준으로 기존 CCTV 자산 활용

> **"CCTV를 새로 사지 않는다. 이미 설치된 카메라를 연결한다."**

ONVIF(Open Network Video Interface Forum)는 IP 기반 영상 감시 기기의 글로벌 표준입니다. 캡스, 한화비전, 하이크비전 등 주요 CCTV 벤더가 모두 지원하므로, 기존 설치된 카메라를 교체하지 않고 그대로 Space OS에 연동할 수 있습니다.

| 항목 | 내용 |
|------|------|
| **표준** | ONVIF Profile S (스트리밍), Profile T (비디오 분석 메타데이터) |
| **스트림** | RTSP (H.264/H.265) → Edge Gateway에서 수신 |
| **Phase 1 활용** | 대시보드 영상 모니터링 + 재실 감지 센서와의 교차 확인 |
| **개인정보 보호** | 영상 저장 없음, 메타데이터(점유 여부/인원 수)만 활용, 안면 인식 미사용 |

---

### 3.5 향후 확장: 비전 AI 연동 가능성 (Phase 2 이후)

> 비전 AI(객체 탐지 모델)는 본 솔루션의 **필수 구성 요소가 아니며**, Phase 1의 핵심 범위에 포함되지 않습니다. 다만, 시스템 아키텍처가 ONVIF 표준 + AWS 클라우드(SageMaker) 위에 설계되어 있으므로, 향후 고도화 단계에서 비전 AI를 자연스럽게 추가할 수 있는 **확장성**을 사전에 확보합니다.

**Phase 2 이후 확장 시나리오 (예시):**
- CCTV 프레임 기반 회의실 인원 카운트 자동화 (재실 센서 보조 수단)
- 비정상 점유 패턴 감지 (장기 방치 물품, 무단 점거 등)
- 공간 사용 히트맵 분석 (좌석 배치 최적화 근거)

이 기능들은 Phase 1의 인프라(Edge Gateway + ONVIF 연동 + Data Lake)가 안정적으로 운영된 이후, 별도의 인프라 투자 없이 소프트웨어 업데이트만으로 활성화할 수 있도록 설계합니다.

## 4. 핵심 기능 요구사항 (Epic 수준)

### 요구사항 우선순위 프레임워크: 카노(Kano) 모델

본 제안은 요구사항의 우선순위를 **카노 모델(Kano Model)**에 따라 분류합니다. 이는 "무엇을 먼저 만들어야 고객 만족이 극대화되는가"를 판단하는 프레임워크입니다.

| 품질 분류 | 정의 | Space OS 적용 |
|-----------|------|-------------|
| **당연적 품질 (Must-Be)** | 없으면 극도로 불만, 있어도 당연시 | 이기종 기기 통합 제어, 통합 대시보드, 무중단 Edge 인프라 |
| **일원적 품질 (Performance)** | 구현 수준에 비례하여 만족도 상승 | CCTV 연동 모니터링, 실시간 알림, 센서 기반 재실 감지 |
| **매력적 품질 (Attractive)** | 없어도 불만 없으나, 있으면 극적 만족 | 어뷰징 자동 탐지 + 자동 과금, 비전 AI 확장 |

### Epic 1. 이기종 IoT 디바이스 통합 관제 대시보드

**카노 분류: 당연적 품질 (Must-Be)** — 이것이 없으면 플랫폼의 존재 이유가 없습니다.

> **목적**: 슈프리마, 에어팝, 캡스, Tuya 등 파편화된 벤더 시스템을 하나의 통합 관제 화면에서 상태 확인 및 제어할 수 있도록 합니다.

| 핵심 기능 | 설명 |
|----------|------|
| **Tuya 기기 자동 프로비저닝** | Tuya C2C API 연동으로 등록된 IoT 기기(조명, 센서, 플러그 등)를 대시보드에 자동 등록 및 원격 제어 |
| **타워램프 실시간 상태 표시** | 회의실별 타워램프 상태(녹/황/적/점멸)를 대시보드에서 실시간 모니터링, 어뷰징 현황 한눈에 파악 |
| **Edge Gateway 헬스 모니터링** | 각 지점 Edge 장비의 CPU, 메모리, 네트워크 상태를 중앙에서 원격 모니터링, 장애 사전 감지 및 알림 |
| **본사/지점 이원화 뷰** | 본사 관제팀은 전 지점 총괄 대시보드, 지점 CM은 해당 지점 운영 대시보드 사용 |

**기대 효과:**
- 5개 벤더 콘솔 → **1개 통합 대시보드**로 관리 복잡도 80% 감소
- 장애 감지 시간: 수 시간(수동 확인) → **수 분 이내(자동 알림)**

---

### Epic 2. ONVIF CCTV 연동 및 센서 기반 재실 감지

**카노 분류: 일원적 품질 (Performance)** — 구현 수준에 비례하여 관리자 만족도가 증가합니다.

> **목적**: 기존 CCTV 자산을 ONVIF 표준으로 연동하고, 재실 감지 센서와 결합하여 회의실의 실시간 점유 현황을 자동 파악합니다.

| 핵심 기능 | 설명 |
|----------|------|
| **ONVIF 카메라 자동 탐색** | Edge Gateway가 동일 네트워크의 ONVIF 호환 카메라를 자동 탐색, 대시보드에 등록 |
| **대시보드 영상 모니터링** | RTSP 스트림을 대시보드에서 실시간 확인 (CM 원격 모니터링용) |
| **재실 감지 센서 연동** | PIR/mmWave 재실 센서를 회의실에 설치, 점유 여부(사용 중/비어 있음)를 실시간 판정 |
| **개인정보보호 준수** | 영상 녹화/저장 없음, 안면 인식 미사용, 메타데이터(점유 여부)만 활용 |

**기대 효과:**
- CM의 회의실 순찰 불필요 → 대시보드에서 **실시간 점유 현황 원격 확인**
- 개인정보보호법 준수 범위 내에서 **데이터 기반 공간 관리** 시작

---

### Epic 3. 회의실 어뷰징 자동 탐지 및 스마트 과금 제어

**카노 분류: 매력적 품질 (Attractive)** — 구현 시 극적인 만족도 향상을 유발합니다.

> **목적**: 재실 센서 데이터와 예약 시스템 데이터를 교차 검증하는 룰 엔진(Rule Engine)으로 어뷰징을 자동 탐지하고, 타워램프 경고 + 자동 과금까지 연결합니다.

| 핵심 기능 | 설명 |
|----------|------|
| **미예약 사용 자동 탐지** | 센서 판정(점유 중) + 예약 시스템(해당 시간 예약 없음) → 타워램프 적색 점등 + CM 알림 + 단계적 경고(조명 점멸 → 차단) |
| **초과 사용 자동 과금** | 예약 종료 후 10분 유예 경과 시 자동 "초과 사용" 판정 → 10분 단위 추가 과금 자동 기록 |
| **노쇼 자동 취소** | 예약 시작 후 15분간 비어 있음 판정 → 예약 자동 취소 + 슬롯 즉시 재공개 + 페널티 자동 적용 |
| **타워램프 물리적 피드백** | 녹색(정상) / 황색(종료 임박) / 적색(어뷰징) / 점멸(즉시 퇴실) 상태로 시각적 경고 |

**룰 엔진 동작 흐름:**

```
재실 센서 (점유 판정)  +  예약 시스템 (예약 상태)
            │                      │
            └──────────┬───────────┘
                       ▼
              ┌─────────────────┐
              │   Rule Engine   │
              │  (교차 검증)     │
              └────────┬────────┘
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   미예약 사용       초과 사용        노쇼
   (적색 점등)      (자동 과금)    (자동 취소)
```

**기대 효과:**
- 과금 누락률: 30% → **0%** (자동 과금)
- 회의실 분쟁 CM 중재: 월 20시간/지점 → **월 2시간/지점** (90% 절감)
- 노쇼 공실 손실: 연간 8,000만원 → **50% 감소**

### 6.3 전략적 효과: 기술 내재화 및 담당자 과부하 해소

#### 기술 파트너의 역할: "외주 개발사"가 아닌 "과부하 분담 파트너"

본 프로젝트의 기술 파트너는 단순 외주 개발을 넘어, 현재 담당자 1인에게 집중된 기술 실행 부담을 직접 분담합니다.

| 영역 | 담당자 부담 (현재) | 파트너 분담 (Phase 1) | **담당자 역할 전환** |

| 플랫폼 구축 | (시도 불가) | 백엔드 + 대시보드 + 룰 엔진 전체 구축 | 운영 시나리오 검증 |
| 벤더 연동 | 벤더별 개별 대응 | Tuya C2C + ONVIF 통합 연동 | 기존 벤더 인터페이스 조율 |
| AI/데이터 | 1인 PoC 시도 | 데이터 파이프라인 설계 + 분석 환경 구축 | 분석 결과 기반 전략적 의사결정 |

> **핵심 메시지**: 담당자 A씨가 "혼자서 모든 것을 실행하는 전문가"에서 **"기술 파트너와 함께 전략적 판단을 내리는 의사결정자"**로 전환됩니다. 이것은 개인의 업무 환경 개선일 뿐 아니라, 1인 의존에 따른 **사업 연속성 리스크를 해소**하는 것이기도 합니다.
