# 운영자 서비스 정의서 (Operator Service Definition)
**공유 오피스 비즈니스 운영 자동화 시스템**

---

## 문서 목적

이 문서는 **공유 오피스 운영사의 비즈니스 효율화**를 위한 시스템 기능을 정의한다.

**핵심 목표:**
- **지점당 운영 인력 0.5명 감축** (60개 지점 기준 연간 인건비 약 9억원 절감)
- **본사-지점 간 정책 배포 시간 90% 단축** (수동 48시간 → 자동 4시간)
- **회의실 초과 이용 수익 누락 제로화** (연간 약 3천만원 추가 매출 확보)

이 문서는 기술이 아닌 **운영자가 하루에 몇 시간을 절약하는가**에 집중한다.

---

## 1. 핵심 페르소나 흐름

### 페르소나 A: HQ 운영 총괄 최본부장 (본사 운영팀장)

**프로필:**
- 전국 60개 지점 총괄 책임자
- 정책 수립 및 전파 담당
- 월간 운영 리포트 작성

---

#### 시나리오 1: 전사 보안 정책 긴급 배포 (월요일 오전)

**상황:** 주말 동안 타 지점에서 보안 사고 발생. 전 지점에 야간 출입 제한 정책을 긴급 적용해야 함.

**기존 방식 (수동):**
1. 정책 문서 작성 (30분)
2. 60개 지점 매니저에게 카카오톡 공지 (20분)
3. 각 지점에서 개별 시스템 설정 변경 (지점당 20분 = 1,200분 = 20시간)
4. 설정 완료 여부 확인 및 독촉 (4시간)
5. **총 소요 시간: 약 25시간 (1일 이상)**

**자동화 방식:**
1. **본사 관리자 포털 접속**
   - "정책 관리" 메뉴 클릭
   - "새 정책 만들기" 버튼

2. **정책 내용 설정**
   - 정책명: "야간 출입 제한 (비상)"
   - 적용 대상: "전체 지점" 체크 (또는 특정 지점 선택)
   - 설정 내용:
     - 출입 허용 시간: 평일 06:00~23:00
     - 야간 출입 가능 등급: "Manager 이상만 허용"
     - 예외 처리: "긴급 상황 시 본사 승인 후 임시 해제 가능"
   - 적용 기간: 2026-02-17 ~ 2026-03-17 (1개월)

3. **배포 설정**
   - 배포 방식 선택:
     - 즉시 배포 (선택)
     - 예약 배포 (날짜/시간 지정)
   - 알림 설정:
     - 지점 매니저에게 카카오톡 자동 발송 체크
     - 입주사에게 앱 푸시 알림 발송 체크

4. **배포 실행 및 모니터링**
   - "전체 배포" 버튼 클릭
   - 배포 진행 상황 실시간 표시:
     - 강남점: 적용 완료 ✓ (3분 소요)
     - 홍대점: 적용 완료 ✓ (2분 소요)
     - 판교점: 적용 중... 45%
     - ...
   - **60개 지점 전체 적용 완료: 15분**

5. **결과 확인**
   - 적용 결과 리포트 자동 생성
   - 미적용 지점 자동 표시 (네트워크 장애 등)
   - 재시도 버튼으로 실패 지점 재배포

**효과:**
- ⏱️ **시간 절감: 25시간 → 15분 (99% 단축)**
- 💰 **인건비 절감: 3명x8시간 → 1명x15분 = 약 19만원 절감 (1회당)**
- 🎯 **정확성: 수동 설정 오류 제로**

**운영자 기대:**
- "클릭 몇 번으로 전 지점 정책 바로 적용"
- "누가 설정 안 했는지 일일이 확인 안 해도 됨"
- "긴급 상황에 빠르게 대응 가능"

---

#### 시나리오 2: 멤버십 등급별 로밍 정책 관리 (월초 정기 작업)

**상황:** 신규 멤버십 상품 출시. "All Access" 고객은 전 지점 출입 가능, "Single Spot" 고객은 계약 지점만 출입 가능하도록 설정.

1. **멤버십 등급 정의**
   - "멤버십 관리" 메뉴 진입
   - 등급 목록:
     - Premium All Access: 전 지점 24시간 출입
     - Standard All Access: 전 지점 평일 출입 (주말 제외)
     - Single Spot: 계약 지점만 출입
     - Hot Desk: 예약 시간에만 출입

2. **로밍 규칙 설정**
   - "Premium All Access" 등급 선택
   - 출입 가능 지점: "전체 60개 지점" 체크
   - 출입 가능 시간: "24시간"
   - 회의실 예약: "당일 예약 가능 (타 지점 포함)"

   - "Single Spot" 등급 선택
   - 출입 가능 지점: "계약 지점만" 선택
   - 타 지점 이용 시: "1회당 2만원 추가 과금 안내 표시"

3. **자동 적용**
   - 저장 버튼 클릭
   - 기존 회원 등급에 따라 자동 권한 업데이트
   - 회원 앱에 변경 사항 푸시 알림 발송

4. **예외 상황 처리**
   - 회원이 타 지점 출입 시도:
     - Single Spot 회원이 홍대점 도어 태그
     - 단말기 화면: "계약 지점이 아닙니다. 1회 이용권 구매 시 출입 가능 (2만원)"
     - 회원 앱에 결제 링크 푸시
     - 결제 완료 시 즉시 문 열림 허용 (1회 한정)

**효과:**
- 💰 **추가 매출 창출: 타 지점 이용 수수료 자동 과금**
- 🎯 **멤버십 차별화: 등급별 혜택 명확화**
- ⚙️ **자동화: 수동 권한 변경 작업 제로**

**운영자 기대:**
- "등급별 혜택 설정하면 시스템이 알아서 처리"
- "타 지점 이용 시 자동으로 과금 가능"
- "고객이 직접 업그레이드 결제 가능 (CS 업무 감소)"

### 페르소나 B: 지점 CM 이매니저 (Community Manager, 강남점)

**프로필:**
- 강남점 현장 운영 매니저
- 입주사 관리, 방문객 응대, 회의실 관리 담당
- 하루 평균 20건 이상 문의 처리

---

#### 시나리오 1: 신규 입주사 온보딩 (목요일 오전)

**상황:** 스타트업 "ABC" 8명이 내일(금요일)부터 입주 예정. 오늘 중으로 출입 권한 설정 완료해야 함.

**기존 방식 (수동):**
1. 입주사로부터 직원 명단 엑셀 수령 (이메일 대기 30분)
2. 명단 확인 및 오타 수정 (20분)
3. 출입 시스템에 8명 개별 등록 (1명당 5분 = 40분)
4. 실물 카드 8장 준비 및 카드 번호 등록 (30분)
5. 카드 활성화 및 테스트 (20분)
6. 입주사에게 카드 전달 및 사용법 교육 (30분)
7. **총 소요 시간: 약 2시간 30분**

**자동화 방식:**
1. **관리자 앱에서 "신규 입주사 등록" 클릭**
   - 회사명: "스타트업 ABC"
   - 계약 기간: 2026-02-21 ~ 2027-02-20 (1년)
   - 멤버십 등급: "Single Spot"
   - 출입 구역: "강남점 5층 Zone-3"
   - 인원: 8명

2. **직원 정보 일괄 입력**
   - 옵션 1: 엑셀 파일 업로드 (이름, 연락처 자동 추출)
   - 옵션 2: 입주사 대표에게 "직원 등록 링크" 전송
     - 대표가 직접 직원 정보 입력 (CM 개입 불필요)

3. **출입 수단 선택**
   - 카드 방식: 실물 카드 8장 발급
     - 카드 리더기에 카드 태그 → 자동 등록
   - 모바일 방식: 직원 연락처로 앱 다운로드 링크 자동 발송
     - 직원이 앱 설치 → 계약 기간 동안 유효한 모바일 키 자동 활성화

4. **자동 권한 설정**
   - 계약 기간 동안만 유효 (2026-02-21 00:00 ~ 2027-02-20 23:59)
   - 계약 종료일 자정에 자동으로 카드/모바일 키 비활성화
   - 출입 가능 구역: 1층 현관, 5층 복도, 5층 Zone-3
   - 회의실 예약 권한 자동 부여

5. **완료 및 알림**
   - 등록 완료 즉시 직원들에게 SMS/카카오톡 발송:
     - "강남점 입주를 환영합니다. 내일부터 출입 가능합니다."
     - 모바일 키 다운로드 링크 포함
   - CM에게 "등록 완료" 알림

**소요 시간: 10분**

6. **자동 퇴실 처리 (계약 종료 시)**
   - 2027년 2월 20일 자정
   - 시스템이 자동으로:
     - 8명 전원 출입 권한 비활성화
     - 회의실 예약 권한 삭제
     - CM에게 "ABC 계약 종료" 알림 발송
     - 입주사 대표에게 "계약 종료, 연장 원하시면 문의하세요" 메시지

**효과:**
- ⏱️ **시간 절감: 2시간 30분 → 10분 (94% 단축)**
- 💰 **인건비 절감: 월 평균 10건 입주 시 약 25시간 절약 = 월 200만원 절감**
- 🎯 **오류 감소: 수동 입력 오류, 권한 설정 누락 제로**
- ⚙️ **자동 만료: 계약 종료 후 출입 불가 (보안 강화)**

**CM 기대:**
- "엑셀만 올리면 자동으로 카드 발급됨"
- "직원들이 직접 앱으로 등록하면 내가 할 일 없음"
- "계약 끝나면 자동으로 권한 사라짐 (일일이 확인 안 해도 됨)"

#### 시나리오 2: 방문객 초대 자동화 (화요일 오후)

**상황:** 입주사 "XYZ"가 내일 오후 2시 미팅에 외부 손님 3명 초대. CM에게 방문자 등록 요청.

**기존 방식 (수동):**
1. 입주사로부터 방문자 명단 카카오톡 수령 (즉시)
2. CM이 출입 시스템에 방문자 3명 수동 등록 (1명당 3분 = 9분)
3. 출입 시간 설정 (내일 14:00~17:00)
4. 일회용 비밀번호 또는 QR 생성
5. 입주사에게 QR 이미지 전달 (카카오톡)
6. 입주사가 손님들에게 재전달
7. **총 소요 시간: 약 15분 (CM 개입 필요)**

**자동화 방식:**
1. **입주사가 직접 앱에서 처리**
   - 입주사 직원이 자기 앱에서 "손님 초대하기" 버튼 클릭
   - 정보 입력:
     - 손님 이름: 김방문, 이고객, 박파트너
     - 방문 날짜: 2026-02-18
     - 방문 시간: 14:00~17:00
     - 방문 목적: 사업 미팅 (선택 사항)
     - 손님 연락처: 010-1234-5678, ...

2. **자동 QR 발급**
   - 시스템이 자동으로 3명분 개별 QR 코드 생성
   - 각 QR 코드는 해당 방문자 전용 (다른 사람 사용 불가)
   - 유효 시간: 2026-02-18 13:55~17:30 (5분 일찍 도착 가능 + 30분 여유)

3. **자동 발송**
   - 손님들에게 SMS/카카오톡 자동 발송:
     - "안녕하세요, FastFive 강남점 방문 초대장입니다."
     - QR 코드 이미지 첨부
     - 주소 및 찾아오는 길 안내
     - "QR 코드를 1층 현관 단말기에 스캔하세요"

4. **CM은 알림만 수신**
   - CM 앱에 "XYZ사가 방문객 3명 등록했습니다" 알림
   - 확인만 하면 됨 (개입 불필요)

5. **방문 당일 자동 처리**
   - 손님이 1층 도착 → QR 스캔 → 문 열림
   - 단말기 화면: "김방문님, 환영합니다. 5층 Zone-3으로 이동하세요"
   - CM에게 "방문객 김방문 입실" 실시간 알림 (선택 사항)

6. **방문 종료 후 자동 무효화**
   - 17:30 이후 QR 자동 만료
   - 손님이 나갈 때는 내부 버튼으로 자유롭게 퇴실

**효과:**
- ⏱️ **시간 절감: 15분 → 0분 (CM 개입 제로)**
- 💰 **인건비 절감: 월 평균 100건 방문자 등록 시 약 25시간 절약 = 월 200만원 절감**
- 🎯 **고객 경험 개선: 입주사가 직접 처리하므로 대기 시간 제로**
- 🔒 **보안 강화: 시간 외 출입 불가, QR 1회용**

**CM 기대:**
- "입주사가 손님 직접 초대하면 나는 할 일 없음"
- "일일이 방문자 명단 받고 등록할 필요 없음"
- "방문 기록은 자동으로 남아서 보안 관리 쉬움"

### 페르소나 C: 빌링 담당 정회계 (본사 재무팀)

**프로필:**
- 회의실 예약 및 초과 이용 요금 관리 담당
- 월말 정산 및 청구서 발행
- 수익 누락 방지 책임

---

#### 시나리오: 회의실 초과 이용 자동 감지 및 과금

**상황:** 강남점 회의실 A는 시간당 2만원. 입주사 "ABC"가 14:00~16:00(2시간) 예약했으나 17:00까지 사용함 (1시간 초과).

**기존 방식 (수동):**
1. 회의실 예약 시스템에서 예약 시간 확인
2. CCTV 또는 CM 육안으로 실제 사용 시간 확인
3. 초과 사용 여부 판단 (애매한 경우 많음)
4. 초과 시간 계산 (1시간)
5. 입주사에게 추가 요금 안내 (전화 또는 메일)
6. 입주사 확인 및 이의 제기 처리
7. 다음 달 청구서에 수동 추가
8. **총 소요 시간: 약 20분/건, 월 50건 시 약 16시간**
9. **수익 누락율: 약 30% (CM이 놓치거나 눈치 보느라 청구 안 함)**

**자동화 방식:**

1. **예약 시 자동 도어락 연동**
   - 입주사 "ABC"가 회의실 A 예약 (14:00~16:00)
   - 시스템이 자동으로:
     - 13:55부터 ABC 직원 카드로 회의실 문 열림 허용
     - 16:00 이후 문 열림 권한 자동 해제

2. **예약 시간 5분 전 알림**
   - 입주사 앱에 푸시: "회의실 A 예약 시작 5분 전입니다"
   - 회의실 문 앞 단말기 화면: "ABC 예약, 13:55부터 입실 가능"

3. **예약 시간 종료 알림**
   - 15:50 (종료 10분 전) 회의실 내부 스피커 안내:
     - "예약 종료 10분 전입니다"
   - 16:00 (종료 시각):
     - 회의실 내부 화면: "예약 시간이 종료되었습니다"
     - "연장 사용 시 10분당 3,500원 추가 과금됩니다"
     - 앱에서 "연장하기" 버튼 표시

4. **초과 사용 자동 감지**
   - 회의실 문 센서 + 재실 감지 센서로 실제 퇴실 시간 자동 측정
   - 16:00 이후에도 센서가 재실 감지 → 초과 사용으로 자동 판정
   - 10분 단위로 자동 카운트

5. **실시간 과금 알림**
   - 16:10 (10분 초과 시):
     - 앱 푸시: "회의실 A 10분 초과 사용 중. 추가 요금 3,500원"
   - 16:20 (20분 초과 시):
     - 앱 푸시: "회의실 A 20분 초과 사용 중. 추가 요금 7,000원"
   - 계속 업데이트

6. **최종 퇴실 시 자동 정산**
   - 17:00 (1시간 초과) 퇴실 감지
   - 시스템이 자동으로:
     - 초과 시간 계산: 60분
     - 추가 요금 계산: 3,500원 x 6회 = 21,000원
     - 입주사 앱에 최종 알림: "회의실 A 1시간 초과 사용. 추가 요금 21,000원이 다음 청구서에 합산됩니다"
     - 빌링 시스템에 자동 기록

7. **월말 자동 청구**
   - 매월 말일 자동 정산
   - "ABC" 청구 내역:
     - 회의실 예약 기본료: 20만원 (월 10회)
     - 초과 사용 추가료: 8만원 (월 4회 초과)
     - 총 청구액: 28만원
   - 청구서 자동 생성 및 이메일 발송

**효과:**
- ⏱️ **시간 절감: 월 16시간 → 0시간 (완전 자동화)**
- 💰 **수익 누락 방지: 30% 누락 → 0% 누락 = 연간 약 3,600만원 추가 매출** (60개 지점 기준)
- 🎯 **분쟁 감소: 센서 기반 객관적 측정으로 이의 제기 90% 감소**
- 😊 **고객 경험: 실시간 알림으로 깜빡 초과 사용 방지**

**빌링 담당 기대:**
- "CCTV 보면서 일일이 확인 안 해도 됨"
- "초과 사용 알아서 측정되고 자동으로 청구됨"
- "입주사도 실시간으로 알림 받으니까 나중에 뭐라 안 함"
- "월말에 클릭 한 번으로 전체 정산 끝"

## 2. 핵심 기능 상세 정의

### 기능 1: 멀티 지점 중앙 관제 (HQ & Branch Logic)

#### A. 정책 중앙 배포 시스템

**기능 설명:**
본사에서 설정한 보안/운영 정책을 전체 또는 선택한 지점에 일괄 배포.

**필요 기능:**
- [ ] **정책 템플릿 생성**
  - 출입 제한 정책 (시간대별, 등급별)
  - 절전 정책 (조명/난방 자동 제어)
  - 비상 정책 (재난 시 자동 문 열림)
  - 멤버십 정책 (등급별 권한)

- [ ] **배포 대상 선택**
  - 전체 지점 일괄 선택
  - 지역별 그룹 선택 (서울권, 경기권 등)
  - 개별 지점 선택 (체크박스)
  - 필터 기능 (예: "회의실 10개 이상 보유 지점만")

- [ ] **배포 방식 설정**
  - 즉시 배포 (지금 바로 적용)
  - 예약 배포 (날짜/시간 지정, 예: 주말 새벽 배포)
  - 단계별 배포 (파일럿 5개 지점 → 검증 → 전체 배포)

- [ ] **배포 진행 모니터링**
  - 실시간 배포 진행률 (60개 중 45개 완료)
  - 실패 지점 자동 표시 (이유: 네트워크 장애, 시스템 점검 중 등)
  - 재시도 기능 (실패 지점만 다시 배포)

- [ ] **배포 결과 리포트**
  - 성공/실패 통계
  - 실패 원인 분석
  - 지점별 적용 시각 기록

---

#### B. 멤버십 등급별 로밍 시스템

**기능 설명:**
회원의 멤버십 등급에 따라 출입 가능한 지점과 시간을 자동으로 제어.

**멤버십 등급 예시:**

| 등급 | 가격 | 출입 가능 지점 | 출입 시간 | 회의실 예약 |
|------|------|---------------|----------|------------|
| Premium All Access | 50만원/월 | 전국 60개 지점 | 24시간 | 무제한 |
| Standard All Access | 35만원/월 | 전국 60개 지점 | 평일 06~23시 | 월 10회 |
| Single Spot | 25만원/월 | 계약 지점만 | 24시간 | 월 5회 |
| Hot Desk | 15만원/월 | 계약 지점만 | 예약 시간만 | 불가 |

**필요 기능:**
- [ ] **등급별 권한 매트릭스 설정**
  - 출입 가능 지점 설정 (전체/지역/개별)
  - 출입 가능 시간대 설정
  - 회의실 예약 권한 및 횟수 제한
  - 타 지점 이용 시 추가 과금 설정

- [ ] **로밍 출입 시 자동 검증**
  - 회원이 타 지점 도어 태그
  - 시스템이 자동으로:
    - 멤버십 등급 확인
    - 출입 허용/거부 판정
    - 거부 시 단말기 화면에 이유 표시 ("계약 지점이 아닙니다")
    - 업그레이드 안내 표시 ("All Access 등급으로 업그레이드 시 이용 가능")

- [ ] **1회 이용권 자동 판매**
  - 출입 거부 시 단말기 화면에 "1회 이용권 구매 (2만원)" 버튼
  - 회원 앱에 결제 링크 푸시
  - 결제 완료 시 당일 해당 지점 출입 권한 즉시 부여
  - 자정에 자동 권한 회수

- [ ] **로밍 이용 통계**
  - HQ 관리자 포털에서 확인:
    - 지점별 타 지점 회원 이용 현황
    - 인기 로밍 지점 순위 (마케팅 활용)
    - 1회 이용권 판매 통계 (추가 매출 확인)

- [ ] **자동 등급 업그레이드 제안**
  - Single Spot 회원이 타 지점 1회 이용권을 월 3회 이상 구매 시:
    - 앱에 자동 알림: "All Access 등급으로 변경하시면 월 6만원 절약됩니다"
    - 클릭 한 번으로 등급 변경 및 결제

### 기능 2: 수익화 및 빌링 연동 (Revenue & Credits)

#### A. 예약-도어락 연동 시스템

**기능 설명:**
회의실/좌석 예약 시간과 도어락/좌석 조명을 자동 연동하여 무단 사용 방지 및 초과 이용 자동 과금.

**필요 기능:**
- [ ] **예약 기반 자동 권한 부여**
  - 회의실 예약 시:
    - 예약 시작 5분 전부터 예약자 카드로 문 열림 허용
    - 예약 종료 시각에 자동으로 권한 해제
    - 무단 입실 방지 (예약 없으면 문 안 열림)

- [ ] **예약 시간 알림**
  - 시작 5분 전: "회의실 A 예약 시작 5분 전입니다" (앱 푸시)
  - 종료 10분 전: 회의실 내부 화면에 "10분 남았습니다" 표시
  - 종료 시각: "예약 시간이 종료되었습니다. 연장 또는 퇴실하세요"

- [ ] **연장 기능**
  - 종료 시각에 앱에서 "30분 연장" 버튼 활성화
  - 연장 가능 여부 자동 확인 (다음 예약 있으면 불가)
  - 연장 요금 자동 계산 및 표시
  - 결제 후 즉시 권한 연장

---

#### B. 초과 이용 자동 감지 및 과금

**기능 설명:**
센서를 통해 실제 이용 시간을 측정하고, 예약 시간 초과 시 자동으로 추가 요금 부과.

**필요 기능:**
- [ ] **재실 감지 센서 연동**
  - 회의실 내부 센서로 실시간 재실 여부 측정
  - 문 센서 + 인체 감지 센서 조합 (정확도 99%)
  - 예약 종료 후에도 재실 감지 시 초과 사용 자동 판정

- [ ] **실시간 초과 시간 카운트**
  - 10분 단위로 자동 카운트
  - 회의실 내부 화면에 "초과 시간: 10분 / 추가 요금: 3,500원" 실시간 표시
  - 앱에도 동일한 정보 푸시

- [ ] **자동 과금 기록**
  - 퇴실 감지 시 최종 초과 시간 계산
  - 추가 요금 자동 계산 (10분 단위 요금 x 초과 구간)
  - 빌링 시스템에 자동 기록
  - 회원 앱에 "초과 이용 내역" 즉시 표시

- [ ] **월말 자동 정산**
  - 매월 말일 자동 정산
  - 입주사별 초과 이용 내역 취합
  - 청구서 자동 생성 (기본료 + 초과료)
  - 이메일 자동 발송

- [ ] **분쟁 방지 기능**
  - 모든 초과 이용 기록에 타임스탬프 저장
  - 센서 데이터 로그 저장 (이의 제기 시 증거 자료)
  - 회원이 앱에서 "초과 이용 내역" 실시간 확인 가능
  - CCTV 영상과 연동 (필요 시 재생 가능)

---

#### C. 수익 분석 대시보드

**필요 기능:**
- [ ] **초과 이용 수익 리포트**
  - 월간 회의실 초과 이용 수익 합계
  - 지점별 초과 이용 수익 순위
  - 회의실별 초과 이용 빈도 (인기 회의실 파악)

- [ ] **1회 이용권 판매 리포트**
  - 타 지점 1회 이용권 판매 건수 및 매출
  - 가장 많이 팔린 지점 (로밍 수요 파악)
  - 등급 업그레이드 전환율

- [ ] **수익 누락 방지 알림**
  - 초과 이용 발생했으나 과금 안 된 케이스 자동 탐지
  - 시스템 오류로 인한 수익 누락 경고

### 기능 3: CM 업무 자동화 (Community Manager Automation)

#### A. 입주 프로세스 자동화

**기능 설명:**
계약 기간과 회원 정보 입력 시 출입 권한 자동 부여 및 계약 종료 시 자동 회수.

**필요 기능:**
- [ ] **계약 기간 기반 자동 권한 관리**
  - 계약 시작일 00:00 자동 권한 활성화
  - 계약 종료일 23:59 자동 권한 비활성화
  - CM이 수동으로 연장/해제할 필요 없음

- [ ] **직원 정보 일괄 등록**
  - 엑셀 파일 업로드 (이름, 연락처, 이메일)
  - 자동 파싱 및 중복 체크
  - 오류 행 자동 표시 (예: 연락처 형식 오류)

- [ ] **입주사 셀프 등록 기능**
  - CM이 입주사 대표에게 "직원 등록 링크" 발송
  - 대표가 직접 직원 정보 입력 (이름, 연락처)
  - 입력 완료 시 CM에게 알림
  - CM은 최종 승인만 클릭 (검토 후 1분 소요)

- [ ] **모바일 키 자동 발급**
  - 직원 등록 완료 시 연락처로 SMS/카카오톡 자동 발송
  - 내용: "입주 환영 메시지 + 앱 다운로드 링크"
  - 직원이 앱 설치 및 인증 시 모바일 키 즉시 활성화

- [ ] **실물 카드 간편 발급**
  - CM이 카드 리더기에 카드 태그
  - 화면에 "등록할 직원 선택" 목록 표시
  - 직원 선택 후 카드 자동 매핑
  - 카드 번호 수동 입력 불필요

- [ ] **계약 종료 자동 처리**
  - 종료일 7일 전: 입주사 대표에게 "계약 종료 안내 + 연장 문의" 메일
  - 종료일 자정: 전체 직원 출입 권한 자동 비활성화
  - 종료일 익일: CM에게 "실물 카드 회수 필요" 알림 (카드 발급한 경우)

---

#### B. 방문객 관리 자동화

**기능 설명:**
입주사가 직접 방문객을 초대하고 QR 발급하는 셀프 서비스. CM 개입 불필요.

**필요 기능:**
- [ ] **입주사 셀프 초대 기능**
  - 입주사 직원이 자기 앱에서 "손님 초대" 버튼
  - 정보 입력: 이름, 연락처, 방문 날짜, 시간
  - 출입 가능 구역 자동 설정 (입주사 구역 + 공용 공간)

- [ ] **QR 자동 발급 및 발송**
  - QR 코드 즉시 생성 (방문자 전용, 1회용)
  - 방문자 연락처로 SMS/카카오톡 자동 발송
  - 내용: 초대장 + QR 이미지 + 찾아오는 길

- [ ] **방문 시간 제한**
  - QR 유효 시간 자동 설정 (예약 시간 ±30분)
  - 시간 외 출입 시도 시 "유효하지 않은 시간입니다" 표시
  - 퇴실은 자유 (나갈 때는 제한 없음)

- [ ] **CM 알림 전용 모드**
  - CM은 "방문객 등록됨" 알림만 수신
  - 승인/거부 불필요 (자동 처리)
  - 필요 시 CM이 방문 기록 조회 가능

- [ ] **방문 기록 자동 저장**
  - 누가 언제 방문했는지 자동 로그
  - 보안 감사 용도로 활용
  - 월간 리포트: "강남점 월간 방문객 300명, 전월 대비 20% 증가"

#### C. CM 대시보드

**기능 설명:**
CM이 지점 운영 상황을 한눈에 파악하고 긴급 상황에 대응할 수 있는 화면.

**필요 기능:**
- [ ] **오늘의 일정 요약**
  - 신규 입주 예정: 2건
  - 계약 종료 예정: 1건
  - 방문객 예정: 8명
  - 회의실 예약: 15건

- [ ] **실시간 현황**

## 5. 핵심 경험 원칙 (운영자 관점)

### 원칙 1: "본사는 한 번에, 지점은 자동으로"
본사에서 정책을 설정하면 60개 지점에 자동 배포.
지점 매니저가 개별 설정할 필요 없음.

### 원칙 2: "입주사가 직접, CM은 승인만"
입주사가 직원 등록, 손님 초대를 직접 처리.
CM은 최종 확인만 하면 됨 (개입 최소화).

### 원칙 3: "시스템이 측정, 자동으로 청구"
초과 이용은 센서가 측정하고 자동 과금.
CM이나 재무팀이 일일이 확인할 필요 없음.

### 원칙 4: "계약 기간이 곧 권한"
계약 시작일에 자동 활성화, 종료일에 자동 비활성화.
CM이 수동으로 권한 변경할 필요 없음.

### 원칙 5: "이상 징후만 알림"
정상 작동 시 알림 없음.
문제 발생 시에만 알림 발송 (알림 피로도 최소화).
