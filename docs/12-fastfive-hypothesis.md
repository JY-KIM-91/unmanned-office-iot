# FastFive Platform Transition Hypothesis

## 1. 목표

외주 출입/IoT 솔루션 의존 구조를
단계적으로 플랫폼 내재화 구조로 전환한다.

---

## 2. 접근 전략

### Phase 1 – 분석
- 현재 벤더 비용 구조 분석
- 기능 분해 (출입 인증 / 로그 / 관리자 페이지 / API / 관제)
- 핵심 기능 vs 부가 기능 분리

### Phase 2 – 최소 내재화
- 출입 인증 서버 레이어 내부화
- 데이터 수집/로그 구조 내부 축적
- 외주 디바이스는 유지

### Phase 3 – 점진적 디바이스 전환
- 신규 지점부터 자체 디바이스 적용
- 점진적 교체 전략

---

## 3. 기대 효과

- 장기 비용 절감
- 데이터 자산 확보
- 지점 확장 시 비용 구조 개선
- 브랜드 기술 역량 강화
